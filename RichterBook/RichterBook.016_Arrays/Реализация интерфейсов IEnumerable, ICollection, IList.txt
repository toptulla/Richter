Базовый класс для массивов реализует интерфейсы коллекций IEnumerable, ICollection и IList. Все эти интерфейсы не
обобщенные. Команда CLR решила не реализовывать их обобщенные версии IEnumerable<T>, ICollection<T> и IList<T>. Т.к.
в их случае возникают проблемы с многомерными массивами, а также с массивами, в которых нумерация начинается не с 0.

Чтобы избежать вышеуказанных проблем разработчики схитрили - при создании 1-го массива начинающегося с 0, CLR
автоматически реализует интерфейсы IEnumerable<T>, ICollection<T> и IList<T>. Кроме того, если тип элемента
массива ссылочный, то таже добавляются реализации IEnumerable<T>, ICollection<T> и IList<T> для всех базовых
типов. Если массив содержит элементы значимого типа, то CLR реализует эти 3-и интерфейса только для этого
значимого типа, для System.ValueType и System.Object эти интерфейсы реализованы не будут.
